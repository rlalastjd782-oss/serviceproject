# Changelog

이 프로젝트는 Semantic Versioning(`MAJOR.MINOR.PATCH`)을 기준으로 관리합니다.

## v1.6.6 - 2026-05-26

- 기록 탭 부위별 분석의 전체 보기에서 특정 부위만 보이던 문제를 완화하기 위해 일간 부위 분석 조회 범위를 넓혔습니다.
- 부위 필터 링크에 섹션 앵커를 추가해 클릭 후 화면이 맨 위로 튀지 않고 부위별 분석 영역에 머물게 했습니다.
- 기록 페이징과 부위별 분석 섹션 사이 간격을 늘렸습니다.
- PWA 캐시와 앱 버전을 `v1.6.6`으로 갱신했습니다.

## v1.6.5 - 2026-05-26

- 부위별 분석 카테고리 필터를 기록 탭의 일간 기록 화면에만 적용되도록 재정리했습니다.
- 기록 화면의 부위 필터와 카드 영역 사이 여백을 조정해 UI가 붙어 보이는 문제를 완화했습니다.
- PWA 캐시와 앱 버전을 `v1.6.5`로 갱신했습니다.

## v1.6.4 - 2026-05-26

- 일간/주간/월간 분석의 부위별 분석 영역에 부위 카테고리 필터를 추가했습니다.
- 선택한 부위만 표시되도록 `part` 쿼리 필터를 적용하고, 필터 변경 시 페이지 번호는 초기화되게 했습니다.
- PWA 캐시와 앱 버전을 `v1.6.4`로 갱신했습니다.

## v1.6.3 - 2026-05-26

- 배포 서버의 UTC 시간대 영향으로 오늘 화면이 한국 시간보다 하루 늦게 잡힐 수 있던 문제를 수정했습니다.
- 앱의 오늘 날짜 계산을 `APP_TIMEZONE` 환경변수 기준으로 바꾸고 기본값을 `Asia/Seoul`로 설정했습니다.
- 날짜가 박히는 첫 화면(`/`, `/?mode=workout`, `/?mode=meal`)을 PWA precache에서 제외해 배포 후 이전 날짜 화면이 남을 가능성을 줄였습니다.
- PWA 캐시와 앱 버전을 `v1.6.3`으로 갱신했습니다.

## v1.6.2 - 2026-05-26

- 설정 비밀번호 PBKDF2 해시 생성/검증을 `health_tracker/security.py`로 옮겨 보안 책임을 한 모듈로 모았습니다.
- 비밀번호 해시 포맷과 오입력 거부를 확인하는 회귀 테스트를 추가했습니다.
- PWA 캐시와 앱 버전을 `v1.6.2`로 갱신했습니다.

## v1.6.1 - 2026-05-26

- CSRF 토큰과 관리자 보호 라우트 기준을 `health_tracker/security.py`로 분리해 `app.py`의 보안 공통 로직 중복을 줄였습니다.
- 전체 소스 점검 과정에서 PWA 캐시와 앱 버전을 `v1.6.1`로 갱신했습니다.

## v1.6.0 - 2026-05-26

- 방문자는 기록/분석을 볼 수만 있고, 수정/삭제/입력/export/import는 관리자만 가능하도록 읽기 전용 모드를 추가했습니다.
- 설정 비밀번호를 관리자 잠금으로 확장하고 민감 GET/export/API와 모든 POST 요청을 관리자 세션으로 보호했습니다.
- 모든 POST 요청에 CSRF 토큰 검증을 추가하고, 일반 form/JSON 운동 시간 저장 요청에 토큰을 붙이도록 했습니다.
- 방문자 화면에서는 작성/수정/삭제 UI를 숨기고 상단에 보기/관리자 상태를 표시합니다.
- PWA 캐시와 앱 버전을 `v1.6.0`으로 갱신했습니다.
