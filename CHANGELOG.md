# Changelog

## v2.6.2 - 2026-05-29

- 오늘 운동 입력, 식단 입력, 주간/월간 분석, 장소 기본 데이터에 남아 있던 깨진 한글 문구를 정상 한국어로 복구했습니다.
- 반복 물음표와 깨진 인코딩 흔적이 사용자 화면 소스에 다시 들어오지 않도록 정적 회귀 테스트를 추가했습니다.

## v2.6.1 - 2026-05-29

- 관리자 대시보드 운영 체크포인트 카드의 깨진 물음표 문구를 정상 한국어로 수정했습니다.
- 사용자 활성, 조치 필요, 데이터 규모 지표 문구가 정상 표시되도록 회귀 테스트를 추가했습니다.

## v2.6.0 - 2026-05-29

- v2.5.12~v2.5.19 고도화 흐름을 묶어 실사용 안정 버전으로 정리했습니다.
- QA 리포트에 주요 페이지 서버 렌더링 시간 측정을 추가했습니다.
- 식단 입력에 2개/3개/4개 빠른 행 추가 버튼을 추가해 여러 식품을 한 번에 입력하기 쉽게 했습니다.
- 식단 빠른 후보 쿼리를 최근 180일 기준으로 제한해 장기 데이터가 쌓였을 때 식단탭 진입 비용을 줄였습니다.
- 기록 검색 결과 CSS를 `records.css`로 분리하고 base layout/service worker precache에 반영했습니다.
- `server*.log`를 git ignore에 추가해 로컬 실행 로그가 작업트리에 섞이지 않게 했습니다.
- 검증: release check, Ruff, compileall, 전체 JS syntax check, 전체 unittest 30개를 통과했습니다.

## v2.5.11 - 2026-05-29

- 오늘 화면을 모드별로 렌더링하도록 정리해 전체/식단 모드에서 운동 입력과 운동 추천 partial을 불필요하게 만들지 않도록 개선했습니다.
- 식단 모드에서는 식단 입력과 기록만 렌더링하고, 운동 모드에서만 세트 빌더/루틴/운동 기록을 렌더링하도록 분리했습니다.
- 주간/월간 분석 화면에 `analysis-action-plan` 카드를 추가해 리포트, 균형, 목표 기준의 다음 행동을 더 잘 보이게 했습니다.
- 관리자 대시보드에 사용률, 기록 없는 계정, 조치 필요 상태를 추가해 계정 관리 판단 지표를 보강했습니다.
- Fold UI 회귀 테스트에 모드별 렌더링과 분석 액션 카드 검증을 추가했습니다.
- 검증: Ruff, compileall, release check, 주요 JS syntax check, 전체 unittest 30개를 통과했습니다.

## v2.5.10 - 2026-05-29

- `app.py`에 남아 있던 분석, 운동, PR, 장소, 요약, 회복, 신체/식단 wrapper를 기능별 facade 모듈로 분리했습니다.

- `app_coaching_reports_facade.py`와 `app_recovery_facade.py`를 분리해 리포트 계산과 회복/추천 코칭 책임을 나눴습니다.
- `app.js`의 부팅/이벤트 바인딩을 `app_boot.js`로 분리하고 base layout과 service worker precache에 반영했습니다.
- `today/index.html`의 전체 보기 패널을 `_overview_panels.html` partial로 분리했습니다.
- 검증: Ruff, compileall, release check, 주요 JS `node --check`, 전체 unittest 30개를 통과했습니다.

## v2.5.9 - 2026-05-29

## 이전 주요 작업 요약

- 로그인/회원가입/관리자 페이지 흐름을 분리했습니다.
- 관리자 계정은 관리자 페이지 중심으로 접근하고, 사용자 계정은 운동/식단/기록 화면을 사용하도록 권한 경계를 강화했습니다.
- 계정별 DB 분리 구조를 적용해 1번 사용자와 2번 사용자의 운동장소/기록이 섞이지 않게 했습니다.
- 장소별 운동장비 관리와 운동 입력 시 장소 기반 장비 후보 노출을 추가했습니다.
- lb 입력 시 kg로 변환 저장하는 기능을 추가했습니다.
- 기록/분석/식단 페이지의 페이지네이션과 일부 모바일 UI를 개선했습니다.
- Galaxy Fold 계열 모바일 폭에서 상단 메뉴, 빠른 메뉴, 운동 입력 UI가 깨지지 않도록 여러 차례 보정했습니다.
