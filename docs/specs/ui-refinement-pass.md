# UI Refinement Pass

## 1. 목적

현재 헬스 트래커의 주요 화면 UI가 사용 흐름에 비해 복잡하고 부적절하게 느껴지는 문제를 개선한다. 이번 작업의 목적은 새 기능을 추가하는 것이 아니라, 기존 화면의 정보 구조, 시각적 위계, 버튼/카드/입력 영역의 일관성을 정리해 사용자가 기록과 확인을 더 빠르게 수행하도록 만드는 것이다.

## 2. 사용자 시나리오

- 사용자는 모바일에서 앱을 열고 오늘 운동, 식단, 요약 중 필요한 작업으로 바로 이동한다.
- 사용자는 운동 중 타이머, 세트 입력, 최근 기록, 현재 운동 목록을 헷갈리지 않고 순서대로 확인한다.
- 사용자는 식단 입력 화면에서 목표, 입력 폼, 오늘 기록을 자연스러운 순서로 본다.
- 사용자는 기록/분석/더보기 화면에서 카드, 필터, 빈 상태, 주요 액션이 같은 규칙으로 보이길 기대한다.
- 사용자는 화면이 과하게 회색이거나 카드가 중첩된 느낌 없이, 필요한 정보만 또렷하게 구분되길 기대한다.

## 3. 주요 화면/상태

- 오늘 요약 화면
  - 날짜 선택, 모드 전환, 오늘 상태, 기록 요약, 데이터 품질, 최근 기록
- 오늘 운동 화면
  - 운동 모드 전환, 집중 모드, 타이머/휴식 타이머, 세트 입력, 현재 운동 목록, 위치/장비, 추천/규칙 카드
- 오늘 식단 화면
  - 식단 목표, 식단 입력, 오늘 식단 기록, 식단 관련 요약
- 기록 화면
  - 일간 기록, 검색, 기간/정렬/필터, 빈 상태, 기록 카드
- 분석 화면
  - 주간/월간/연간/PR/운동별/장비별 분석 카드와 현재 범위 표시
- 더보기 화면
  - 캘린더, 주간 계획, 장소, 식단 템플릿, 데이터 센터 등 보조 기능 진입 카드
- 공통 상태
  - 기본 상태, 선택/활성 상태, 입력 중 상태, 저장 중/저장 완료 상태, 오류/빈 상태, 모바일 좁은 화면

## 4. 기능 요구사항

- 화면별 최우선 사용자 행동을 명확히 보이게 한다.
  - 오늘 운동: 타이머와 운동 입력 흐름을 우선 배치한다.
  - 오늘 식단: 목표와 입력, 오늘 기록을 순서대로 배치한다.
  - 기록/분석: 기간과 현재 범위, 결과 목록을 먼저 이해할 수 있게 한다.
- 카드 중첩감을 줄인다.
  - 바깥 섹션만 명확한 카드로 보이게 하고, 내부 요소는 flat row, chip, badge, 입력 컨트롤로 정리한다.
  - 카드 안에 또 다른 카드가 반복되는 구조는 디자인 차원에서 피한다.
- 색상 방향을 정리한다.
  - 전체를 회색으로 덮는 방향은 피한다.
  - 주요 섹션은 깨끗한 흰색 또는 밝은 표면을 유지하고, 내부 보조 요소만 옅은 회색/블루 계열로 낮춘다.
  - 활성 버튼, 선택 탭, 주요 CTA는 주변 요소보다 명확해야 한다.
- 간격과 밀도를 맞춘다.
  - 모바일 기준으로 섹션 간격, 카드 내부 padding, 리스트 row 간격을 일정하게 유지한다.
  - 화면 상단 메뉴와 세그먼트 컨트롤은 줄바꿈/삐져나감 없이 한 손 조작이 가능해야 한다.
- 텍스트 위계를 정리한다.
  - 섹션 제목, 보조 설명, 숫자 지표, 버튼 라벨의 크기와 굵기를 화면별로 통일한다.
  - 긴 한글 라벨은 모바일에서 겹치거나 잘리지 않도록 줄바꿈 또는 축약 규칙을 둔다.
- 상태 표현을 강화한다.
  - 현재 선택된 탭/필터/기간은 즉시 알아볼 수 있어야 한다.
  - 빈 상태는 다음 행동 버튼 또는 안내 문구와 함께 제공한다.
  - 저장 완료/오류 상태는 입력 흐름을 방해하지 않으면서 확인 가능해야 한다.
- 공통 컴포넌트 규칙을 정리한다.
  - 버튼, 세그먼트 컨트롤, 필터 바, 카드, badge, chip, 입력 필드는 화면마다 같은 시각 규칙을 따른다.
  - 오늘 화면만 예외적으로 다른 스타일이 되지 않도록 기록/분석 화면의 안정적인 패턴을 참고한다.

## 5. 예외 케이스

- 데이터가 없는 신규 사용자
  - 빈 카드만 나열하지 않고, 바로 기록할 수 있는 주 액션을 제공한다.
- 운동 기록이 많은 사용자
  - 최근 기록/현재 운동 목록이 길어져도 타이머와 입력 영역이 밀려나지 않아야 한다.
- 식단/운동 중 하나만 기록한 날짜
  - 없는 영역은 빈 상태로 처리하고, 다른 영역의 요약을 방해하지 않는다.
- 모바일 좁은 화면
  - 상단 메뉴, 날짜 선택, 필터, CTA가 화면 밖으로 넘치지 않아야 한다.
- PWA 캐시
  - UI 변경 후 이전 CSS가 남아 보이지 않도록 릴리즈 버전과 service worker cache 갱신이 필요하다.
- 접근 제한/읽기 전용 사용자
  - 쓰기 액션은 비활성 또는 안내 상태로 보이고, 읽기 가능한 정보 구조는 유지한다.

## 6. 완료 조건

- 오늘 요약/운동/식단 화면에서 카드 중첩감과 과한 회색 표면이 줄어든다.
- 운동 화면에서 타이머, 휴식 타이머, 운동 입력, 현재 기록 목록의 순서가 자연스럽다.
- 주요 탭/세그먼트/필터의 활성 상태가 명확하다.
- 기록/분석/더보기 화면의 카드, 필터, 빈 상태가 동일한 디자인 규칙을 따른다.
- 모바일 화면에서 텍스트, 버튼, 필터가 겹치거나 화면 밖으로 넘치지 않는다.
- 변경 후 주요 화면 캡처 또는 PDF 기준으로 디자이너/기획자가 비교 검토할 수 있다.

## 7. 개발 참고사항

- 이번 문서는 구현 방식 지정이 아니라 UI/UX 요구사항 정리 문서다.
- 기존 `UI_QA.md`와 `UI_QA_FINDINGS_2026-05-30.md`의 오늘 화면 이슈를 우선 참고한다.
- 최근 변경 이력상 전체 회색화는 부정적 피드백이 있었으므로, 흰색 섹션 카드와 옅은 내부 보조 표면을 기본 방향으로 본다.
- CSS 수정 시 마지막 override만 누적하기보다 실제 원본 규칙과 최종 override의 책임을 함께 정리하는 것이 좋다.
- PWA 환경이므로 manifest, service worker cache, `VERSION` 갱신 필요 여부를 개발 단계에서 확인한다.
- 가정: 사용자가 말한 "UI가 부적절하다"는 현재까지 기록된 오늘 화면/공통 카드 체계 불만을 포함하는 전반 UI 다듬기 요청으로 해석했다.
- 가정: 이번 작업의 1차 우선순위는 오늘 화면이며, 기록/분석/더보기는 공통 UI 규칙 정합성 확인 범위로 포함한다.

## 8. QA 체크포인트

- 모바일 390px 안팎 너비에서 오늘 요약/운동/식단의 첫 화면과 긴 스크롤 캡처를 확인한다.
- 오늘 운동 화면에서 타이머와 입력 영역 사이에 불필요한 카드가 끼어들지 않는지 확인한다.
- 오늘 요약 화면의 데이터 품질, 최근 기록, 요약 카드가 과도한 중첩 카드처럼 보이지 않는지 확인한다.
- 활성 탭, 선택 필터, 주요 CTA가 비활성 요소와 충분히 구분되는지 확인한다.
- 빈 상태와 오류 상태가 카드 색상/간격/버튼 규칙을 깨지 않는지 확인한다.
- 기록/분석/더보기 화면에서 공통 카드와 필터 스타일이 오늘 화면과 충돌하지 않는지 확인한다.
- 긴 한글 텍스트가 버튼, badge, chip, 카드 안에서 잘리거나 겹치지 않는지 확인한다.
- 변경 후 정적 자산 테스트, 주요 UI navigation 테스트, release check를 실행할 수 있도록 개발 handoff에 남긴다.
