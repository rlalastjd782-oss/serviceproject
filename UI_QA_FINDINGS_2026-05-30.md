# UI QA Findings - 2026-05-30

## 목적

사용자가 제공한 26개 UI 캡처 이미지를 기준으로, 현재 화면에서 이상해 보이는 부분과 개선 방향을 정리한다.

이번 문서는 **분석/기록용 문서**이며, 코드 수정/커밋/푸시는 진행하지 않았다.

## 전체 공통 문제

1. 상단 네비게이션이 모바일 폭에서 2줄로 밀리면서 일부 메뉴가 잘린다.
   - 예: `더보기`, 버전 배지, 로그아웃 버튼이 오른쪽에서 잘리거나 화면 밖으로 나간다.
   - 개선: 모바일에서는 상단 메뉴를 2줄 grid로 설계하거나, 하단 탭/가로 스크롤 탭 중 하나로 명확히 통일해야 한다.

2. 카드와 폼 컨테이너의 폭 기준이 서로 다르다.
   - 어떤 영역은 좌우 여백이 있고, 어떤 영역은 화면 끝까지 붙어 보인다.
   - 특히 버튼/입력칸/리스트 카드가 같은 페이지 안에서도 폭이 다르게 보인다.
   - 개선: `section`, `form`, `list-toolbar`, `card`의 max-width, padding, gap 기준을 하나로 정해야 한다.

3. 전역 CSS override가 너무 넓게 적용되어 페이지별 UI가 무너졌다.
   - `tool-form`, `filter-form`, `list-toolbar`, `month-picker-form-wide` 같은 공통 클래스가 서로 다른 용도의 화면에 동일하게 적용되고 있다.
   - 결과적으로 식단 템플릿, 기록 검색, 운동 라이브러리, 분석 필터 화면에서 레이아웃이 어긋난다.
   - 개선: 공통 클래스에 직접 디자인을 입히기보다 페이지/컴포넌트별 래퍼 클래스로 범위를 좁혀야 한다.

4. 카드 안에 카드가 반복되어 층층이 쌓인 느낌이 난다.
   - 기록 카드 내부의 badge, 필터 박스 내부의 select, 분석 카드 내부의 progress row가 모두 그림자/배경을 가져 피라미드처럼 보인다.
   - 개선: 바깥 카드만 입체감을 주고, 내부 요소는 flat pill 또는 단순 row로 낮춰야 한다.

5. 어두운 섹션 헤더가 너무 자주 반복된다.
   - 페이지마다 섹션 헤더가 모두 차콜 박스로 보이면서 화면이 무겁고 단조롭다.
   - 개선: 최상위 섹션만 차콜 헤더를 쓰고, 하위 섹션은 밝은 카드 헤더 또는 텍스트 헤더로 내려야 한다.

6. 버튼 색상과 상태 표현이 일관되지 않다.
   - 활성 탭은 어두운 버튼인데, 일부 CTA도 같은 어두운 버튼이라 현재 위치와 행동 버튼이 구분되지 않는다.
   - 개선: 탭 활성, 기본 버튼, 주요 CTA, 위험 버튼의 색상 역할을 분리해야 한다.

7. 빈 데이터 화면에서 카드가 과도하게 커 보인다.
   - 값이 `0`인 카드들이 세로로 길게 반복되어 실제 정보량보다 화면을 많이 차지한다.
   - 개선: 빈 상태에서는 요약 카드를 compact layout으로 전환하거나 empty state를 먼저 보여줘야 한다.

8. 모바일에서 입력 필드와 버튼 높이가 페이지마다 다르다.
   - 어떤 입력은 40px, 어떤 입력은 카드 전체 높이처럼 보인다.
   - 개선: 모바일 form control 높이, padding, border 기준을 고정해야 한다.

9. 문서/화면 일부 한글이 깨져 있다.
   - `NOTES.md`, `CHANGELOG.md`는 실제 한글이 깨진 상태다.
   - 일부 캡처에서도 텍스트가 정상 한글로 보이지 않는 흔적이 있다.
   - 개선: 파일 인코딩을 UTF-8로 정리하고, 깨진 문서는 재작성해야 한다.

## 화면별 문제 및 개선 방향

### 01_login_user.png

- 상단 헤더의 버전 배지가 오른쪽에서 잘린다.
- 네비게이션 메뉴가 2줄로 내려가면서 `더보기`가 오른쪽에서 잘린다.
- 로그인/회원가입 전 화면이라면 메뉴 전체를 노출할 필요가 없어 보인다.
- 개선:
  - 로그인 전 화면에서는 앱 메뉴를 숨기고 로그인/가입/미리보기만 노출한다.
  - 헤더 메타 영역은 모바일에서 줄바꿈되거나 숨김 처리한다.

### 02_preview.png

- 미리보기 화면의 큰 제목이 오른쪽에서 잘린다.
- hero 영역 텍스트 폭과 카드 폭이 맞지 않는다.
- 하단 기능 카드들은 흰 배경 카드로 갑자기 분위기가 달라진다.
- 개선:
  - 미리보기 전용 레이아웃을 일반 앱 레이아웃과 분리한다.
  - 제목은 모바일에서 2~3줄로 자연스럽게 줄바꿈되도록 max-width와 font-size를 조정한다.

### 03_today_overview.png

- 날짜 선택 영역의 좌우 화살표가 카드 밖에 둥둥 떠 있는 느낌이다.
- `운동/식단/전체` 세그먼트와 상단 날짜 컨트롤의 간격이 불균형하다.
- 분석 신뢰도 카드에서 원형 그래프와 설명 카드가 서로 다른 높이로 어색하다.
- 개선:
  - 날짜 컨트롤은 하나의 toolbar 안에 `이전 / 날짜 / 이동 / 다음` 구조로 통일한다.
  - 분석 신뢰도는 ring + 텍스트 + 세부 지표를 하나의 grid로 재배치한다.

### 04_today_workout.png

- 장소/장비 영역과 다음 액션 카드가 모두 어두운 헤더를 가지고 있어 구분이 과하다.
- 장비가 없을 때 빈 상태 카드가 너무 크게 보인다.
- `현재 장소`, `장비` 라벨과 입력/버튼의 간격이 좁고 텍스트 위계가 약하다.
- 개선:
  - 장소 섹션은 compact control panel로 낮추고, 어두운 헤더는 제거 또는 축소한다.
  - 장비 없음 상태는 작은 안내 pill로 처리한다.

### 05_today_meal.png

- 목표 달성률 카드들이 세로로 길게 쌓여 화면을 많이 차지한다.
- progress bar 오른쪽 값이 화면 밖으로 잘리는 느낌이다.
- 전체적으로 식단 탭인데 운동 목표까지 같은 카드 구조로 보여 정보 구분이 약하다.
- 개선:
  - 목표 달성률은 2열 compact 카드 또는 accordion으로 정리한다.
  - 목표 진행률 값은 progress bar 안/아래로 고정 배치한다.

### 06_records_daily.png

- 기간 버튼 `7일/14일/30일/60일/90일`이 전부 세로로 쌓여 공간을 과하게 사용한다.
- 필터 박스 내부에 다시 박스가 들어가 피라미드 느낌이 난다.
- `정렬`, `표시` 라벨이 세로로 쪼개져 보인다.
- 개선:
  - 기간 버튼은 horizontal segmented control로 유지하거나 2열 grid로 제한한다.
  - 정렬/표시 select는 label + select 한 줄 구조로 다시 잡는다.

### 07_records_search.png

- `newest`가 그대로 노출되어 사용자용 한글 UI와 맞지 않는다.
- 검색 결과/정렬/페이지 카드의 높이와 간격이 서로 다르다.
- 상세 필터 input들이 2열로 보이다가 화면 폭에서 잘릴 위험이 있다.
- 개선:
  - 정렬 값은 내부 코드값이 아니라 `최신순` 같은 label만 보여야 한다.
  - 검색 필터는 모바일에서 완전 1열로 통일한다.

### 08_analysis_weekly.png

- 주 선택 영역에서 label, input, 버튼, 화살표가 한 컨테이너 안에서 정렬되지 않는다.
- `보기` 버튼이 너무 강한 CTA로 보여 탭 활성 버튼과 충돌한다.
- 주간 분석 카드의 버튼 `백업/PR/운동별/장비별`은 기능 버튼인지 탭인지 애매하다.
- 개선:
  - 기간 선택 컴포넌트를 `이전 / 현재 기간 / 다음` + 보조 버튼 구조로 재설계한다.
  - 분석 이동 버튼은 subnav와 같은 스타일 또는 secondary link로 낮춘다.

### 09_analysis_monthly.png

- 월 선택 영역도 주간과 같은 문제가 있다.
- 주요 지표 카드들이 아래에서 잘리며 카드 높이가 일관되지 않아 보인다.
- 개선:
  - 월/주/연도 선택 UI를 하나의 공통 `period-control` 컴포넌트로 통합한다.

### 10_analysis_yearly.png

- 연도 선택 카드의 input과 버튼이 서로 균형이 맞지 않는다.
- 연간 리포트 CTA 버튼 `전년 비교/JSON/운동 CSV/식단 CSV`가 탭처럼 보인다.
- 개선:
  - export 성격 버튼은 작은 ghost button으로 낮추고, 주요 분석 탭과 시각적으로 분리한다.

### 11_analysis_exercises.png

- 운동 상세 select가 비어 보이거나 너무 넓게 잡힌다.
- 과부하 분석 카드가 빈 상태인데 큰 어두운 헤더가 반복된다.
- 부위 밸런스 카드는 갑자기 원색 배경이라 현재 회색 테마와 튄다.
- 개선:
  - 운동 선택 select는 빈 상태 텍스트를 명확히 표시한다.
  - 과부하 분석 empty state는 compact 안내 카드로 낮춘다.
  - 부위 밸런스 색상은 원색 대신 low-saturation pastel 또는 badge 색으로 제한한다.

### 12_analysis_pr.png

- PR 필터 영역이 카드처럼 보이지만 내부 요소 간격이 일정하지 않다.
- `0개 운동`, `0개 최근 PR` 영역과 하단 지표 카드의 위계가 약하다.
- 개선:
  - PR 페이지는 hero summary + filter panel + rank list로 구역을 명확히 나눈다.

### 13_analysis_equipment.png

- 장비 scope 버튼 `이번 주/이번 달/전체`가 세로로 쌓이며 너무 큰 공간을 차지한다.
- 정렬/표시 컨트롤이 다시 피라미드 구조로 보인다.
- 개선:
  - scope 버튼은 3분할 segmented control로 고정한다.
  - result toolbar는 모든 페이지에서 같은 한 줄 또는 2열 패턴으로 통일한다.

### 14_meals_weekly.png

- 주간 식단 상단의 날짜 선택 영역이 좌우로 잘린다.
- 요약 카드가 0값일 때 너무 큰 세로 카드로 반복된다.
- 개선:
  - 식단 주/월 화면은 분석 화면과 같은 기간 선택 컴포넌트를 사용한다.
  - 데이터 없음 상태에서는 큰 요약 카드 대신 empty summary를 먼저 노출한다.

### 15_meals_monthly.png

- 월간 식단도 주간 식단과 동일하게 날짜 컨트롤이 잘린다.
- 목표 요약 헤더가 화면 중간부터 등장하며 위계가 어색하다.
- 개선:
  - 월간/주간 식단의 상단 구조를 동일하게 정리한다.

### 16_more.png

- More 페이지 첫 카드에는 어두운 헤더가 없고, 아래 섹션에는 어두운 헤더가 있어 톤이 섞인다.
- More 메뉴 항목들이 리스트인지 카드인지 애매하다.
- 개선:
  - More 홈은 `요약 카드` + `기능 그룹 카드` 구조로 명확히 나눈다.
  - 각 메뉴 항목은 동일한 height, padding, hover/active 스타일을 적용한다.

### 17_locations.png

- 장소 관리 화면에서 통계 카드가 세로로 길게 쌓인다.
- 새 장소 추가 form의 label/input 구분은 되지만 전체 폭과 버튼 폭이 맞지 않는다.
- 체크박스가 일반 입력 필드와 같은 행 흐름에 녹지 않는다.
- 개선:
  - 통계는 3개를 compact grid로 보여준다.
  - 장소 추가 form은 모바일 1열, desktop 2열로 정리한다.
  - 체크박스는 작은 option row로 따로 분리한다.

### 18_action_insights.png

- 개인화 다음 운동 헤더 우측 badge가 잘린다.
- 날짜 조회 영역의 버튼과 input 폭이 다르다.
- 후보가 없을 때 카드 안 빈 안내와 버튼이 반복되어 무겁다.
- 개선:
  - 우측 badge는 모바일에서 헤더 아래로 내려야 한다.
  - 개인화 후보 없음 상태는 단일 empty card로 정리한다.

### 19_record_check.png

- 분석 신뢰도 ring과 설명 영역은 괜찮지만, 아래 체크 항목들이 비슷한 높이로 반복되어 정보 위계가 약하다.
- `정리 우선순위` 영역의 `7건` pill이 빈 행처럼 보인다.
- 개선:
  - record check는 quality summary, action priority, issue list 세 구역으로 분리한다.
  - action count는 제목 오른쪽 badge로만 표시한다.

### 20_calendar.png

- 캘린더 날짜 칸은 정돈되었지만 전체 칸이 너무 동일해서 기록 있는 날/없는 날 구분이 약하다.
- 요일 헤더가 보이지 않거나 약하다.
- 개선:
  - 요일 헤더를 고정 노출하고, 기록 있는 날에만 작은 dot/색상 accent를 준다.
  - 빈 날짜는 더 옅게, 오늘 날짜는 별도 표시한다.

### 21_plate_calculator.png

- 계산 결과 pill이 흰색/검은 테두리라 현재 테마와 다르게 튄다.
- 원판 계산과 워밍업 세트 카드 내부가 모두 선으로 구획되어 딱딱하다.
- 개선:
  - 결과 pill은 회색 gradient pill로 통일한다.
  - input/result를 별도 카드가 아니라 같은 tool card 내부 row로 정리한다.

### 22_data_center.png

- 우측 badge가 헤더에서 잘린다.
- 최근 백업 파일명이 카드 안에서 줄바꿈되며 너무 크게 보인다.
- 점검 항목 리스트가 단순 박스 반복으로 보여 우선순위가 약하다.
- 개선:
  - 긴 파일명은 ellipsis 또는 2줄 clamp 처리한다.
  - 상태 요약은 compact grid, 점검 항목은 checklist row로 바꾼다.

### 23_meal_templates.png

- 상단 메뉴가 2줄로 밀리고 `더보기`가 잘린다.
- 식단 템플릿 페이지의 적용 날짜 form에서 오른쪽 버튼이 잘린다.
- 저장된 템플릿/최근 식단 섹션이 비어 있는데도 어두운 헤더가 반복되어 무겁다.
- 개선:
  - 빈 섹션은 어두운 헤더 대신 empty state 카드만 보여준다.

## 우선 정리해야 할 CSS 구조

### 1. 위험한 전역 selector

아래 selector들은 여러 페이지를 동시에 망가뜨릴 가능성이 높다.

- `.tool-form`
- `.filter-form`
- `.record-list-toolbar`
- `.list-toolbar`
- `.list-tools`
- `.compact-select`
- `.month-picker-form-wide`
- `.period-picker`
- `.analysis-action-grid`
- `.yearly-action-grid`
- `.settings-action-row`
- `.detail-row`
- `.record-summary .badge`

개선 방향:

- 이 selector들에 직접 강한 `display`, `grid-template-columns`, `width: 100%`, `box-shadow`를 넣지 않는다.
- 페이지별 래퍼에서만 스타일을 제한한다.
  - 예: `.records-page .filter-form`
  - 예: `.analysis-page .period-control`
  - 예: `.library-section .library-filter-form`

### 2. 컴포넌트 기준 재정의 필요

아래 컴포넌트를 먼저 정의하고, 페이지별로 재사용해야 한다.

- `app-header`
- `top-nav`
- `section-card`
- `section-title-bar`
- `form-panel`
- `period-control`
- `result-toolbar`
- `metric-card`
- `data-card`
- `flat-pill`
- `empty-state`

### 3. 모바일 기준

모바일 기준은 다음으로 고정하는 것이 좋다.

- 본문 좌우 padding: 10~12px
- 섹션 간 gap: 12~16px
- 버튼 높이: 38~42px
- input 높이: 40~44px
- 카드 radius: 10~12px
- 내부 pill radius: 999px
- 내부 pill에는 강한 그림자 금지
- 바깥 카드에만 soft shadow 허용

## 개선 작업 권장 순서

1. 현재 `ui_rebuild_04.css`에 쌓인 최근 전역 override를 감사한다.
2. `tool-form`, `filter-form`, `list-toolbar`, `period-picker`의 전역 grid 강제 규칙을 제거하거나 범위를 축소한다.
3. 상단 헤더/네비게이션 모바일 구조를 먼저 고정한다.
4. 기간 선택 컴포넌트를 주간/월간/연간/식단/캘린더에서 통일한다.
5. 기록/검색/운동별/장비별 result toolbar를 하나의 컴포넌트로 통일한다.
6. 카드 내부 badge/pill은 그림자를 제거하고 flat 처리한다.
7. 빈 데이터 화면용 compact empty state를 만든다.
8. 전체 스크린샷을 다시 캡처해서 26개 화면을 재검수한다.

## 결론

현재 문제는 단순 색상 문제가 아니라 **전역 CSS override가 너무 넓고, 컴포넌트 기준 없이 페이지마다 덮어쓴 결과**다.

다음 작업은 개별 화면을 계속 땜질하는 것보다, 공통 레이아웃 컴포넌트와 페이지별 scope를 먼저 정리하는 방향이 맞다.
